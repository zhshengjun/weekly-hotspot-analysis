# 总结提示词

用于生成《本周热点分析》的摘要、分类分析和核心结论。输出中文，避免空话。面向政务研究材料时，采用稳健、正式、可进入政策简报的表述。

## 单条资讯摘要

对每条资讯生成 `ai_summary` 时使用：

```text
请基于以下资讯，写一段 80 到 150 字中文摘要。
必须覆盖三点：发生了什么、为什么重要、对“{领域}”的影响。
不要只改写标题，不要加入原文没有支持的判断。
如领域涉及浙江交通、物流、基础设施、流量经济或“六张网”，优先说明其对浙江规划、项目推进、网络协同、降本增效或政策争取的意义。

标题：{title}
来源：{source_name}
发布时间：{published_at}
正文要点：{content_points}
```

## 分类分析

每个分类开头写 1 段综合分析时使用：

```text
请基于以下同一分类资讯，写一段 150 到 250 字中文分类分析。
说明本类资讯共同反映的趋势、关键变化和可能影响。
不要逐条复述标题；要提炼共同主题。

领域：{领域}
分类：{category}
资讯列表：{items}
```

## 核心摘要

报告开头的核心摘要使用：

```text
请基于本周全部资讯，生成 3 到 6 条中文核心摘要。
每条摘要聚焦一个明确结论，格式为短句，不超过 60 字。
优先覆盖政策变化、产业趋势、重大项目、公司动态和风险信号。政务研究场景下，结论应能支撑“形势判断、问题识别、工作建议”的后续写作。

领域：{领域}
统计周期：{开始日期} 至 {结束日期}
分类与资讯：{classified_items}
```

## 总体判断

如需要报告结尾，可使用：

```text
请基于本周资讯，写一段 120 到 200 字中文总体判断。
回答：本周该领域最重要的变化是什么？接下来最值得关注的变量是什么？
只基于已列资讯，不做投资、法律、医疗等直接建议。
```

## 本周态势判断

生成总体判断前，先基于已入选资讯做轻量态势识别。只能引用已入选资讯，不允许根据常识或外部猜测补判断。

```text
请基于以下已入选资讯，判断本周“{领域}”是否出现明显态势。

可选态势：
- 政策窗口期：政策、监管、标准、规划、通知等资讯占比较高，或同一方向政策连续发布。
- 项目推进密集期：项目、投资、招标、中标、开工、投产、签约等资讯集中出现。
- 产业活跃期：企业、产品、融资、合作、订单、技术发布等资讯集中出现。
- 风险暴露期：处罚、事故、整改、争议、预警、投诉、召回等资讯集中出现。
- 常态跟踪期：以上特征都不明显。

输出 2 到 4 句话：
1. 给出态势标签。
2. 说明判断依据，必须点出对应资讯类型或代表性事件。
3. 说明下一步最值得关注的变量。

领域：{领域}
统计周期：{开始日期} 至 {结束日期}
已入选资讯：{items}
```

## 报告自检与修正

输出最终报告前使用：

```text
请检查以下《本周热点分析》草稿，只修正问题部分，不要重写全文。

检查项：
1. 是否包含标题、统计周期、核心摘要、本周焦点、分类热点分析、总体判断。
2. 每条资讯是否在正文中包含可点击来源名称、发布时间和领域相关性。
3. 分类是否控制在 3 到 6 个；是否存在同一资讯进入多个分类。
4. 本周焦点是否为 1 到 3 条，并说明入选原因。
5. 态势判断是否由已入选资讯支撑；没有证据的判断必须删除或降级为“常态跟踪期”。
6. 是否存在搜索结果页、转载页、无发布时间资讯或无来源预测。

如果发现问题：
- 缺少固定层级：补齐该层级。
- 分类过散或过粗：重新合并或拆分分类。
- 判断无证据：删除或降级，不要补造依据。
- 来源或发布时间缺项：补齐已入选资讯正文中的来源链接和发布时间；无法补齐则从正文移除。

草稿：{draft}
已入选资讯：{items}
```

## 禁止事项

- 不要编造数据、发布日期、来源或链接。
- 不要把“值得关注”“影响深远”等空话当结论。
- 不要把政务研究材料写成投资建议、营销文案或媒体评论。
- 不要输出没有证据支撑的预测。
- 不要把同一条资讯写进多个分类。
- 不要只因出现一条政策、项目或风险资讯就判断进入窗口期；必须有集中度或代表性事件支撑。
