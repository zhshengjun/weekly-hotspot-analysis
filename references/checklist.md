# 交付前检查清单

用于《本周热点分析》交付前最后一遍自检。P0 未通过不得交付；P1 尽量修正，无法修正时在正文中降级说明；P2 只影响表达质量。

## P0 可信度门槛

- 每条资讯都有原始来源名称、可点击原文链接和明确发布时间。
- 发布时间落在统计周期内；周期外信息只作为背景，不进入入选资讯。
- 来源链接不是搜索结果页、聚合页、官网首页、转载首页或栏目首页。
- 同一事件只出现一次；多源报道保留最原始或最权威来源。
- 核心摘要、本周焦点、分类分析和总体判断都能被已入选资讯支撑。
- 不编造标题、发布时间、来源、链接、统计数据或政策名称。
- 默认交付必须真实生成 HTML 文件，并通过文件存在检查。
- 默认 HTML 必须保留 `assets/report-template.html` 的样式锚点：`hero`、`section-heading`、`section-banner`、`timeline-item`、`date-badge`、`source-line`、`window.print()`。
- PDF 只在用户明确要求时生成；PDF 交付必须真实生成 PDF 文件，并通过文件存在检查。

## P1 结构门槛

- 报告包含标题、统计周期、核心摘要、本周焦点、分类热点分析、总体判断。
- 核心摘要 3 到 6 条；本周焦点 1 到 3 条；分类 3 到 6 个。
- 每个分类先写综合分析，再列入选资讯。
- 分类名称贴合用户领域，不使用无意义的大而空栏目。
- 高频领域合格资讯不足时，先完成语义扩展和多入口补搜，再说明本周可核验资讯偏少。

## P2 表达门槛

- AI 摘要说明发生了什么、为什么重要、对该领域的影响。
- 政务研究场景使用稳健正式文风，不写投资建议或营销话术。
- 不输出候选统计、搜索入口统计、剔除池或来源明细表，除非用户明确要求。
- 结尾判断写下一步关注变量，不写无证据预测。
